# 海特博试云试验检测手簿软件

## 项目概述

海特博试云试验检测手簿软件是一款专为试验检测机构打造的综合性管理系统，实现了从委托接收到报告出具的全流程数字化管理。项目通过现代化技术栈重构，大幅提升了系统性能和用户体验，为试验检测工作提供了高效、规范的数字化解决方案。

## 旧版本分析

### 技术栈
- **前端技术**：JSP (Java Server Pages)、JavaScript、jQuery
- **后端技术**：Java、Servlet、JSP
- **开发方式**：传统的后端渲染方式，前后端耦合度高
- **文件结构**：基于目录结构的页面组织，功能模块分散

### 技术特点
- 采用JSP技术，结合后端Java代码实现页面渲染
- 使用传统的jQuery进行前端交互，缺乏组件化思想
- 页面结构清晰，但维护成本高，扩展性差
- 数据交互主要通过表单提交和页面刷新实现

### 亮点
- 系统功能完善，涵盖了试验检测的各个环节
- 业务逻辑处理成熟，满足试验检测机构的核心需求
- 页面布局合理，操作流程符合行业习惯
- 数据管理能力强，支持复杂的业务逻辑处理

## 新版本分析

### 技术栈
- **前端框架**：Vue 3 + TypeScript + Composition API
- **构建工具**：Vite 6.2.4 + pnpm 10.7.1
- **UI库**：Ant Design Vue
- **状态管理**：Pinia
- **路由**：Vue Router 4
- **网络请求**：Axios + 自定义API封装
- **样式**：TailwindCSS + Less
- **测试框架**：Vitest 3.1.1 + Vue Test Utils
- **其他工具**：@vueuse/core、echarts、three.js、qrcode等

### 技术特点
- **组件化开发**：采用Vue 3的Composition API和<script setup>语法，实现组件的复用和逻辑的清晰分离
- **类型安全**：使用TypeScript，提高代码质量和可维护性
- **现代化构建**：使用Vite构建工具，提供更快的开发和构建速度
- **响应式设计**：结合TailwindCSS，实现响应式布局，支持多设备访问
- **状态管理**：使用Pinia，提供简洁、强大的状态管理能力
- **API封装**：自定义IlisApiHelper，统一处理API请求和响应
- **自动导入机制**：配置自动导入Vue、@vueuse/core、Pinia及组件，简化开发流程
- **后端代理开发**：实现跨域请求和会话管理，提升开发环境体验

### 核心技术实现
1. **装饰器系统**：设计和实现装饰器系统（@FormField、@TableField、@CustomField等），实现实体类字段的声明式配置
2. **业务组件库**：开发IlisTable、IlisFormSearch、IlisContainer等核心业务组件，封装表格、表单、搜索等通用功能
3. **Hooks系统**：实现useTableHooks、useStateHooks、useTableHeight等核心Hooks，封装业务逻辑
4. **表格处理**：自定义useTableHooks，封装表格的分页、排序、选择、导出等功能
5. **API管理**：统一的API封装，支持不同类型的请求和响应处理
6. **状态管理**：使用Pinia实现全局状态管理，支持持久化存储
7. **单元测试**：集成Vitest单元测试框架，编写关键组件和工具函数的测试用例

### 亮点
- **现代化技术栈**：全面采用Vue 3 Composition API和<script setup>语法，提升开发效率和代码可维护性
- **装饰器系统**：实现声明式字段配置，简化实体类定义，减少重复代码
- **组件化开发**：封装丰富的业务组件和Hooks，实现业务逻辑复用，提高开发效率
- **自动导入机制**：配置自动导入Vue、@vueuse/core、Pinia及组件，简化开发流程
- **后端代理开发**：实现跨域请求和会话管理，提升开发环境体验
- **单元测试覆盖**：集成Vitest框架，保证代码质量和稳定性
- **新旧系统平滑过渡**：确保业务功能的连续性，降低系统升级风险
- **性能优化**：使用Vite构建工具，实现快速的开发和构建体验
