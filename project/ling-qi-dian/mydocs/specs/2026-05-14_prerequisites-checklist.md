## 按阶段可用性速查

| 阶段 | 需要准备就绪的事项 | 可暂缺（不阻塞） |
|---|---|---|
| **Phase 0** 工程骨架 | P0 全部 | 全部 P2/P3 |
| **Phase 1** 设计文档 | P0 全部 | P2/P3 |
| **Phase 2-F** 前端开发 | P0 全部 + Phase 1 产出（API 契约文档） | P2/P3（用 Mock 完全独立跑通） |
| **Phase 2-B** 后端开发 | P0 全部 + P2-1~P2-4（uniCloud） + P2-5~P2-9（公众号认证） + P2-10~P2-18（收钱吧凭证） | P3 |
| **Phase 3** 联调部署 | **全部就绪** | 无 |

### 关键结论

- 🔥 **前端开发（Phase 2-F）不依赖任何第三方审核**：基于 API 契约 + Mock 数据即可独立完成全部页面与流程
- 🔥 **后端开发（Phase 2-B）需要 uniCloud 账号 + 公众号 AppID + 收钱吧凭证**，但这些都是在线注册，无审核等待
- 🔥 **ICP 备案（P3-3）是唯一不可控的阻塞项**：10-20 天，必须在 Phase 0 启动时就提交，否则上线会等
