# 会员体系设计文档 v5.0

> 本文档是机车俱乐部饮品与社区系统的会员体系完整设计，涵盖会员等级、充值规则、积分规则。
> 本文档与 `db-schema.md`、`api-contracts.md` 共同构成 Phase 1 设计契约层。
>
> **版本历史**：
>
> - v1.0：初始版本（基于累计消费升级）
> - v2.0：增加游戏化体系（签到/任务/裂变）
> - v3.0：改为积分制升级，机车圈等级名称
> - v4.0：最终版本（见习→青铜→白银→黄金→钻石→至尊，积分门槛调整）
> - v4.1：拆分成长值与积分，成长值仅用于等级，积分可用于抵扣
> - v4.2：7级等级体系，机车特色权益（洗车/保养/救援/备用车），充值档位优化
> - v5.0：**仅充值升级体系**，普通用户无等级，分品类折扣，开业活动¥680直达青铜

***

## 一、会员等级体系

### 1.1 等级定义（6级）

| 等级值 | 等级名称 | 累计充值门槛（平时） | 开业活动门槛 | 积分倍率 | 等级图标建议 |
|:---:|:---|---:|---:|---:|:---|
| 0 | **普通用户** | 无 | - | 1.0x | 👤 普通用户 |
| 1 | **青铜** | ¥1,000 | **¥680** ⭐ | 1.1x | 🥉 青铜骑士 |
| 2 | **白银** | ¥3,000 | - | 1.2x | 🥈 白银骑士 |
| 3 | **黄金** | ¥8,000 | - | 1.3x | 🥇 黄金骑士 |
| 4 | **铂金** | ¥15,000 | - | 1.5x | ⭐ 铂金骑士 |
| 5 | **钻石** | ¥30,000 | - | 1.8x | 💎 钻石骑士 |
| 6 | **至尊** | ¥50,000 | - | 2.0x | 👑 至尊骑士 |

> **核心规则**：
> - 仅**累计充值金额**决定等级，消费不升级
> - 普通用户无等级，消费仅获1.0x积分，无折扣
> - 等级**永久保留**，无降级，无续期要求
> - 开业活动期间（30天），首充¥680即可达青铜（平时需¥1,000）

### 1.2 等级计算规则

```javascript
function calculateLevel(totalRecharge, isOpeningActivity = false) {
  if (totalRecharge >= 50000) return 6;  // 至尊
  if (totalRecharge >= 30000) return 5;  // 钻石
  if (totalRecharge >= 15000) return 4;  // 铂金
  if (totalRecharge >= 8000) return 3;   // 黄金
  if (totalRecharge >= 3000) return 2;   // 白银
  if (totalRecharge >= 1000) return 1;   // 青铜（平时）
  if (isOpeningActivity && totalRecharge >= 680) return 1;  // 青铜（开业活动）
  return 0;  // 普通用户
}
```

## 二、会员充值规则

### 2.1 平时充值档位（5档）

| 档位ID | 充值金额 | 赠送余额 | 赠送比例 | 赠送积分 | 可达等级 | 设计理由 |
|:---:|:---:|---:|---:|---:|:---|:---|
| 1 | ¥198 | ¥20 | 10% | 218积分 | 无等级 | 低门槛体验，让犹豫用户先体验储值便利 |
| 2 | ¥680 | ¥68 | 10% | 748积分 | 无等级（平时）| 接近青铜门槛，开业活动期间直达青铜 |
| 3 | ¥1,000 | ¥100 | 10% | 1,100积分 | **青铜** ⭐ | 正式会员门槛，享全部青铜权益 |
| 4 | ¥3,000 | ¥300 | 10% | 3,300积分 | **白银** | 忠实用户，解锁洗车服务 |
| 5 | ¥8,000 | ¥800 | 10% | 8,800积分 | **黄金** | 高价值用户，配件开始打折 |

> **说明**：
> - ¥198档：虽无等级但享赠送，降低首次尝试门槛
> - ¥680档：平时仅获赠送，开业活动期间首充直达青铜
> - 高档位：满足高消费用户需求，快速升级享更多权益

### 2.2 开业活动充值档位（30天限时）

| 档位ID | 充值金额 | 赠送余额 | 赠送积分 | 额外赠送 | 可达等级 | 活动说明 |
|:---:|:---:|---:|---:|---:|:---|:---|
| 1 | ¥680 | ¥68 | 748积分 | +100积分 | **青铜** ⭐ | 首充直达青铜（省¥320）|
| 2 | ¥1,000 | ¥100 | 1,100积分 | +200积分 | 青铜 | 标准档位，额外赠送 |
| 3 | ¥3,000 | ¥300 | 3,300积分 | +600积分 | 白银 | 忠实用户档位 |
| 4 | ¥8,000 | ¥800 | 8,800积分 | +1,600积分 | 黄金 | 高价值用户档位 |

> **开业活动规则**：
> - 活动时间：开业后30天
> - 首充¥680即达青铜（平时需¥1,000）
> - 额外赠送积分（约1.1倍）
> - 活动结束恢复平时档位

### 2.3 充值处理流程

```
1. 用户选择充值档位
2. 调用云函数 member-recharge
3. 创建充值订单，调用支付接口
4. 支付成功回调
5. 更新余额：balance += 充值金额 + 赠送余额
6. 发放积分：points += 赠送积分
7. 检查等级：调用 member-level-check（基于累计充值）
8. 写入充值记录：recharge_logs
9. 写入积分流水：point_logs
10. 返回结果给前端
```

## 三、积分规则

### 3.1 积分获取规则

| 获取方式 | 普通用户 | 青铜 | 白银 | 黄金 | 铂金 | 钻石 | 至尊 |
|:---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **充值积分** | 充值金额×1.0 | 充值金额×1.1 | 充值金额×1.2 | 充值金额×1.3 | 充值金额×1.5 | 充值金额×1.8 | 充值金额×2.0 |
| **消费积分** | 实付金额×1.0 | 实付金额×1.1 | 实付金额×1.2 | 实付金额×1.3 | 实付金额×1.5 | 实付金额×1.8 | 实付金额×2.0 |
| **签到积分** | 5积分/天 | 5积分/天 | 10积分/天 | 10积分/天 | 15积分/天 | 15积分/天 | 20积分/天 |
| **分享积分** | 10积分/次 | 10积分/次 | 15积分/次 | 15积分/次 | 20积分/次 | 20积分/次 | 30积分/次 |

> **二期活动积分**（上线后补充）：
> - 连续签到奖励（7天/30天额外奖励）
> - 分享裂变奖励（邀请好友注册/充值）
> - 任务系统奖励（完善资料、首次发帖等）

### 3.2 积分计算示例

**场景1：青铜会员充值¥1,000**
```
充值积分：1,000 × 1.1 = 1,100积分
赠送积分：1,100积分（档位赠送）
合计获得：2,200积分
```

**场景2：黄金会员消费¥500（实付）**
```
消费积分：500 × 1.3 = 650积分
```

**场景3：钻石会员月度积分获取**
```
消费¥2,000：2,000 × 1.8 = 3,600积分
签到30天：15 × 30 = 450积分
分享4次：20 × 4 = 80积分
月合计：4,130积分 ≈ ¥41.3抵扣
```

### 3.3 积分使用规则

| 使用场景 | 规则 | 说明 |
|:---|:---|:---|
| 订单抵扣 | 100积分 = 1元 | 基础抵扣比例 |
| 单次抵扣上限 | 订单金额的30% | 比原来20%提高，增加积分价值感 |
| 最低使用门槛 | 100积分起用 | 降低使用门槛 |
| 积分商城 | 兑换商品/优惠券 | 二期上线 |
| 积分抽奖 | 消耗积分抽奖 | 二期上线 |

### 3.4 积分有效期

- 获得之日起 **12个月**
- 优先使用即将过期的积分（FIFO）
- 过期积分自动清零，不做其他处理
- 到期前30天推送提醒通知

### 3.5 积分流水记录

**point_logs 集合**：

```json
{
  "_id": "ObjectId",
  "member_id": "用户ID",
  "change": 150,
  "after_points": 1250,
  "reason": "消费奖励",
  "reason_code": "consume",
  "order_id": "订单ID",
  "expire_time": "2027-05-21T00:00:00Z",
  "create_time": "2026-05-21T10:00:00Z"
}
```

**reason_code 枚举**：
- `recharge`: 充值赠送
- `consume`: 消费奖励
- `check_in`: 签到奖励（二期）
- `share`: 分享奖励（二期）
- `task`: 任务奖励（二期）
- `exchange`: 积分兑换（二期）
- `lottery`: 积分抽奖（二期）
- `expire`: 积分过期
- `deduct`: 订单抵扣

## 七、用户场景示例

### 场景1：新用户首次到店（被开业活动吸引）

```
消费¥50（饮品）→ 看到开业活动"充值¥680直达青铜会员"
→ 充值¥680 → 到账¥748 + 848积分（含额外赠送）
→ 成为青铜会员 → 后续消费享饮品9.8折 + 1.1x积分
```

### 场景2：新用户暂不充值，纯消费

```
消费¥50 → 获得50积分（1.0x）
→ 消费¥500 → 获得500积分
→ 累计550积分，可抵扣¥5.5
→ 但始终无折扣，体验差
→ 看到其他会员享折扣，产生充值意愿
```

### 场景3：青铜会员持续消费

```
青铜会员（充值¥1,000）→ 每月消费¥500
→ 每月获得积分：500×1.1 = 550积分
→ 签到30天：5×30 = 150积分（二期）
→ 分享4次：10×4 = 40积分（二期）
→ 月合计：740积分 ≈ ¥7.4抵扣
→ 年合计：8,880积分 ≈ ¥88.8抵扣 + 生日礼
```

### 场景4：高价值用户快速升级

```
直接充值¥8,000 → 到账¥8,800 + 8,800积分
→ 成为黄金会员
→ 享饮品9折 + 工时费8.5折 + 配件9.8折
→ 每月2次免费洗车 + 50公里免费救援
→ 消费积分1.3x，快速累积积分
```

## 八、版本变更总结

### v5.0 核心变更

| 变更项 | v4.2 | v5.0 |
|:---|:---|:---|
| **升级方式** | 成长值（充值+消费+签到）| 仅充值升级 |
| **等级数量** | 7级（见习→至尊）| 6级（普通用户→至尊）|
| **非充值用户** | 见习等级（消费可升级）| 普通用户（无等级）|
| **等级有效期** | 12个月需续期 | 永久保留，无降级 |
| **充值门槛** | ¥98起 | ¥198起，开业活动¥680直达青铜 |
| **折扣体系** | 统一折扣率 | 分品类折扣（饮品/工时费/配件）|
| **积分倍率** | 1.0x~1.8x | 1.0x~2.0x（普通用户固定1.0x）|

### 优化目标达成

| 目标 | 达成情况 |
|:---|:---|
| 提高商业价值 | ✅ 强制充值享权益，筛选高价值用户 |
| 降低运营复杂度 | ✅ 无成长值计算，无续期逻辑 |
| 提升用户心智 | ✅ 等级=充值身份，简单明了 |
| 保护核心利润 | ✅ 配件单独折扣规则 |
| 降低首充门槛 | ✅ 开业活动¥680直达青铜 |
| 提升用户粘性 | ✅ 二期签到/分享/任务积分激励 |
