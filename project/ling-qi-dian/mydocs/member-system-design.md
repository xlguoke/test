### 1.3 等级计算规则

**升级规则**：

- 根据 `members.total_points_earned`（累计获得积分）自动计算
- 每次获得积分时实时检查并更新等级
- 升级即时生效，推送升级通知

**降级规则**：

- 等级永久有效，不降级
- 但等级权益需定期续期（见1.4节）

**等级计算公式**（基于成长值）：

```javascript
function calculateLevel(growthPoints) {
  if (growthPoints >= 50000) return 6;  // 至尊
  if (growthPoints >= 30000) return 5;  // 钻石
  if (growthPoints >= 15000) return 4;  // 铂金
  if (growthPoints >= 8000) return 3;   // 黄金
  if (growthPoints >= 3000) return 2;   // 白银
  if (growthPoints >= 1000) return 1;   // 青铜
  return 0;                              // 见习（0-99）
}
```

## 二、会员充值规则

### 2.1 充值档位（3档）

| 档位ID | 充值金额（元） | 赠送余额（元） | 赠送比例 | 赠送成长值    | 赠送积分    | 可达等级 | 设计理由               |
| ---- | ------- | ------- | ---- | -------- | ------- | ---- | ------------------ |
| 1    | ¥98     | ¥10     | 0  | 100成长值   | 100积分   | 见习 ✅ | 低门槛体验，适合新客首次尝试（接近青铜） |
| 2    | ¥980    | ¥98     | 10%  | 1,000成长值 | 1,000积分 | 青铜 ✅ | 中等储值，直接成为正式会员，享9.8折 |
| 3    | ¥2,980  | ¥298    | 10%  | 3,000成长值 | 3,000积分 | 白银 ✅ | 大额储值，享9.5折+每月洗车服务 |

> **说明**：
>
> - 仅设置3个充值档位，降低用户决策门槛
> - 充值同时赠送**成长值**（用于等级升级）和**积分**（用于消费抵扣）
> - 成长值 = 充值金额 × 1.0，积分 = 充值金额 × 1.0
> - 等级升级采用成长值标准，充值不是唯一途径
> - 充值¥98达到见习（100成长值），再消费约¥900或再充值可达青铜（1,000成长值）
> - 不充值用户可通过消费约¥100达到见习，消费约¥1,000达到青铜
> - 开业活动为独立模块，不在会员体系中实现

### 2.2 充值规则

- 单笔最低 ¥98，最高 ¥50,000
- 支付成功后即时到账
- 充值金额不可退款，赠送金额不可提现
- 充值记录写入 `recharge_logs` 集合
- 成长值和积分分别写入对应流水集合

### 2.3 充值处理流程

```
1. 用户选择充值档位
2. 调用云函数 member-recharge
3. 创建充值订单，调用支付接口
4. 支付成功回调
5. 更新余额：balance += 充值金额 + 赠送余额
6. 发放成长值：growth_points += 赠送成长值
7. 发放积分：points += 赠送积分
8. 检查等级：调用 member-level-check（基于 growth_points）
9. 写入充值记录：recharge_logs
10. 写入成长值流水：growth_logs
11. 写入积分流水：point_logs
12. 返回结果给前端
```

## 三、成长值与积分规则

### 3.1 概念区分

| 概念      | 用途        | 获取方式        | 消耗方式      | 有效期  |
| ------- | --------- | ----------- | --------- | ---- |
| **成长值** | 仅用于等级计算   | 充值、消费、签到、任务 | 不可消耗      | 永不过期 |
| **积分**  | 用于消费抵扣、兑换 | 充值、消费、签到、任务 | 抵扣消费、兑换商品 | 12个月 |

### 3.2 成长值获取

| 获取方式  | 计算方式          | 说明           |
| ----- | ------------- | ------------ |
| 充值成长值 | 充值金额 × 1.0    | 充值时一次性获得     |
| 消费成长值 | 实付金额（元）× 1.0  | 仅实付金额，余额消费不给 |
| 签到成长值 | 5成长值/天 + 连续奖励 | 二期上线         |
| 任务成长值 | 按任务奖励         | 二期上线         |

> **注意**：成长值仅用于等级计算，不可用于消费抵扣。

### 3.3 积分获取

| 获取方式 | 计算方式          | 说明           |
| ---- | ------------- | ------------ |
| 充值积分 | 充值金额 × 1.0    | 充值时一次性获得     |
| 消费积分 | 实付金额（元）× 等级倍率 | 仅实付金额，余额消费不给 |
| 签到积分 | 5积分/天 + 连续奖励  | 二期上线         |
| 任务积分 | 按任务奖励         | 二期上线         |

**等级倍率**（仅实付金额计算积分）：

| 等级 | 消费积分倍率 | 示例（实付¥100） |
| -- | ------ | ---------- |
| 见习 | 1.0x   | 100积分      |
| 青铜 | 1.1x   | 110积分      |
| 白银 | 1.2x   | 120积分      |
| 黄金 | 1.3x   | 130积分      |
| 钻石 | 1.5x   | 150积分      |
| 至尊 | 1.8x   | 180积分      |

> **重要说明**：
>
> - 消费积分仅针对**实付金额**（微信支付、支付宝等）计算
> - 使用**余额支付**的部分**不获得消费积分**
> - 混合支付时（余额+微信），仅微信实付部分计算积分

### 3.4 成长值与积分计算示例

**场景1：用户充值 ¥1,500**

| 项目     | 成长值       | 积分        | 说明         |
| ------ | --------- | --------- | ---------- |
| 充值获得   | +1,500    | +1,500    | 充值金额 × 1.0 |
| **合计** | **1,500** | **1,500** | 直接升至青铜会员   |

**场景2：青铜会员消费 ¥100（微信支付）**

| 项目     | 成长值     | 积分      | 说明                    |
| ------ | ------- | ------- | --------------------- |
| 消费获得   | +100    | +110    | 成长值=实付×1.0，积分=实付×1.1x |
| **合计** | **100** | **110** | -                     |

**场景3：用户先充值再消费（完整流程）**

用户充值 ¥1,500 → 消费 ¥100（微信支付）：

| 步骤     | 动作          | 成长值变化     | 积分变化      | 成长值余额 | 积分余额  |
| ------ | ----------- | --------- | --------- | ----- | ----- |
| 1      | 充值 ¥1,500   | +1,500    | +1,500    | 1,500 | 1,500 |
| 2      | 消费 ¥100（微信） | +100      | +110      | 1,600 | 1,610 |
| 3      | 使用积分抵扣 ¥5   | -         | -500      | 1,600 | 1,110 |
| **最终** | -           | **1,600** | **1,110** | -     | -     |

> **关键说明**：
>
> - 充值和消费都会分别获得成长值和积分，**不重复、不冲突**
> - 成长值：充值1.0x + 消费1.0x = **2个渠道独立累计**
> - 积分：充值1.0x + 消费1.0\~1.8x = **2个渠道独立累计**
> - 使用积分抵扣消费时，**只减少积分，不影响成长值**

### 3.5 积分使用

| 使用场景   | 规则         | 说明       |
| ------ | ---------- | -------- |
| 订单抵扣   | 100积分 = 1元 | 基础抵扣比例   |
| 单次抵扣上限 | 订单金额的20%   | 防止全额抵扣   |
| 最低使用门槛 | 500积分起用    | 降低小额抵扣频率 |
| 积分兑换   | 积分商城商品     | 二期上线     |
| 积分抽奖   | 抽奖消耗积分     | 二期上线     |

### 3.6 积分有效期

- 获得之日起 12 个月
- 优先使用即将过期的积分（FIFO）
- 过期积分不做任何处理（已从可用积分中扣除）
- 到期前30天推送提醒通知

### 3.7 积分流水记录

**point\_logs 集合**：

```json
{
  "_id": "ObjectId",
  "member_id": "用户ID",
  "change": 150,
  "after_points": 1250,
  "reason": "消费奖励",
  "reason_code": "consume",
  "order_id": "订单ID",
  "expire_time": "2027-05-21T00:00:00Z",
  "create_time": "2026-05-21T10:00:00Z"
}
```

**reason\_code 枚举**：

- `consume`: 消费奖励
- `recharge`: 充值赠送
- `check_in`: 签到奖励（二期）
- `task`: 任务奖励（二期）
- `exchange`: 积分兑换（二期）
- `lottery`: 积分抽奖（二期）
- `expire`: 积分过期
- `deduct`: 订单抵扣

## 七、业务规则

### 7.1 折扣计算规则

```javascript
// 订单折扣计算
function calculateDiscount(orderAmount, memberLevel, benefitsStatus) {
  // 获取等级折扣率
  const discountRate = getDiscountRate(memberLevel);
  
  // 权益降级检查
  if (benefitsStatus === 'expired') {
    // 使用下一等级折扣率
    const actualRate = getDiscountRate(memberLevel - 1);
    return orderAmount * actualRate;
  }
  
  return orderAmount * discountRate;
}
```

### 7.2 积分计算规则

```javascript
// 消费积分计算
function calculateConsumePoints(orderAmount, memberLevel) {
  const multiplier = getPointsMultiplier(memberLevel);
  return Math.floor(orderAmount * multiplier);
}
```

### 7.3 充值处理规则

```javascript
// 充值处理
async function processRecharge(memberId, tierId) {
  const tier = getRechargeTier(tierId);
  const isActivityPeriod = checkActivityPeriod();
  
  // 计算赠送积分
  let bonusPoints = tier.rechargeAmount; // 1.0倍
  
  // 更新余额
  const totalBalance = tier.rechargeAmount + tier.bonusBalance;
  
  // 更新积分
  const newPoints = currentPoints + bonusPoints;
  const newTotalEarned = totalPointsEarned + bonusPoints;
  
  // 检查等级
  const newLevel = calculateLevel(newTotalEarned);
  
  // 写入记录
  await saveRechargeLog(memberId, tier, bonusPoints);
  await savePointLog(memberId, bonusPoints, 'recharge');
  await updateMember(memberId, { balance, points, totalPointsEarned, level });
  
  return { newBalance, newPoints, newLevel };
}
```

### 7.4 权益续期检查规则

```javascript
// 权益续期检查（定时任务，每日执行）
async function checkBenefitsExpiry() {
  const members = await getMembersWithExpiringBenefits();
  
  for (const member of members) {
    const requirement = getBenefitsRequirement(member.level);
    const pointsIn12Months = await getPointsEarnedInLast12Months(member.id);
    
    if (pointsIn12Months < requirement) {
      // 权益降级
      await updateMember(member.id, {
        benefitsStatus: 'expired',
        level: member.level - 1 // 权益降为下一等级
      });
      await sendBenefitsExpiryNotification(member.id);
    }
  }
}
```

## 八、注意事项

### 8.1 金额单位

- 所有金额字段统一使用**分**（整数）存储
- 前端展示时转换为元（除以100）
- 积分使用整数存储，无小数

### 8.2 并发安全

- 充值、积分发放、等级更新需在事务中执行
- 使用数据库事务或乐观锁防止并发问题
- 积分流水必须与余额变更同步

### 8.3 定时任务

| 任务     | 执行时间     | 说明                               |
| ------ | -------- | -------------------------------- |
| 积分过期检查 | 每日 00:00 | 处理过期积分，更新 total\_points\_expired |
| 权益续期检查 | 每日 02:00 | 检查会员权益是否需降级                      |
| 生日礼发放  | 每日 08:00 | 当月生日会员发放饮品券                      |

### 8.4 数据一致性

- 积分流水必须与 members.points 保持一致
- total\_points\_earned = total\_points\_used + total\_points\_expired + points
- 定期校验数据一致性
