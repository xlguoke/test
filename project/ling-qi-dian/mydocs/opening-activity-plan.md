# 开业活动计划文档 v1.0

> 本文档是机车俱乐部饮品与社区系统的开业活动独立计划，涵盖活动规则、技术实现、开发排期。
> **重要**：开业活动为独立模块，在主体程序（Phase 0-3）开发完成后再添加，不混入核心会员体系。

---

## 一、活动概述

### 1.1 活动名称
**「开业狂欢 · 充值翻倍」**

### 1.2 活动时间
- 活动周期：开业后30天
- 可配置：活动开始时间、结束时间通过后台管理配置

### 1.3 活动目标
- 快速积累首批会员
- 提升初始充值转化率
- 制造开业热度与话题

---

## 二、活动规则

### 2.1 核心规则

| 规则项 | 说明 |
|-------|------|
| 活动时间 | 开业后30天（可后台配置） |
| 活动内容 | 充值赠送积分1.5倍 |
| 参与对象 | 所有会员 |
| 叠加规则 | 与其他活动不叠加 |

### 2.2 充值档位（活动期）

| 档位 | 充值金额 | 赠送余额 | 赠送成长值 | 赠送积分（1.5倍） | 对应等级 |
|-----|---------|---------|-----------|-----------------|---------|
| 1 | ¥98 | ¥10 | 100成长值 | 150积分 | 见习 ✅ |
| 2 | ¥980 | ¥98 | 1,000成长值 | 1,500积分 | 青铜 ✅ |
| 3 | ¥2,980 | ¥298 | 3,000成长值 | 4,500积分 | 白银 ✅ |

### 2.3 与正常期对比

| 档位 | 正常期赠送积分 | 活动期赠送积分 | 额外赠送 |
|-----|-------------|--------------|---------|
| 1 | 100积分 | 150积分 | +50积分 |
| 2 | 1,000积分 | 1,500积分 | +500积分 |
| 3 | 3,000积分 | 4,500积分 | +1,500积分 |

## 三、技术实现方案

### 3.1 数据库变更

**新增集合**：`activities`（活动配置表）

```json
{
  "_id": "ObjectId",
  "activity_code": "opening_recharge_2026",
  "activity_name": "开业狂欢·充值翻倍",
  "activity_type": "recharge_bonus",
  "status": 1,
  "start_time": "2026-06-01T00:00:00Z",
  "end_time": "2026-07-01T00:00:00Z",
  "bonus_multiplier": 1.5,
  "bonus_type": "points",
  "description": "开业期间充值赠送积分1.5倍",
  "create_time": "2026-05-21T00:00:00Z",
  "update_time": "2026-05-21T00:00:00Z"
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|-----|------|------|
| activity_code | String | 活动编码（唯一） |
| activity_name | String | 活动名称 |
| activity_type | String | 活动类型（recharge_bonus=充值赠送） |
| status | Number | 0未开始/1进行中/2已结束/-1已取消 |
| start_time | Date | 活动开始时间 |
| end_time | Date | 活动结束时间 |
| bonus_multiplier | Number | 赠送倍率（1.5=1.5倍） |
| bonus_type | String | 赠送类型（points=积分） |

**索引**：
- `{ activity_code: 1 }` unique
- `{ status: 1, start_time: 1, end_time: 1 }` 活动查询

### 3.2 云函数变更

**新增云函数**：

| 云函数 | 说明 |
|-------|------|
| `activity-list` | 获取当前进行中的活动列表 |
| `activity-detail` | 获取活动详情 |
| `activity-check` | 检查活动是否有效（内部调用） |

**修改云函数**：

| 云函数 | 修改内容 |
|-------|---------|
| `member-recharge-tiers` | 增加活动信息返回（活动期展示活动档位） |
| `member-recharge-callback` | 增加活动倍率计算逻辑 |

### 3.3 充值处理流程（活动期）

```
1. 用户选择充值档位
2. 调用云函数 member-recharge
3. 创建充值订单，调用支付接口
4. 支付成功回调
5. 检查活动：调用 activity-check
   - 活动进行中 → bonus_multiplier = 1.5
   - 无活动 → bonus_multiplier = 1.0
6. 计算赠送积分：bonusPoints = rechargeAmount × bonus_multiplier
7. 更新余额：balance += rechargeAmount + bonusBalance
8. 发放积分：points += bonusPoints
9. 更新累计获得积分：total_points_earned += bonusPoints
10. 检查等级：调用 member-level-check
11. 写入充值记录：recharge_logs（增加 activity_code 字段）
12. 写入积分流水：point_logs
13. 返回结果给前端
```

### 3.4 前端变更

**会员中心页面**：
- 增加活动入口/活动倒计时展示
- 活动期展示活动专属充值档位

**充值页面**：
- 活动期展示活动横幅（"开业狂欢·充值翻倍"）
- 展示活动倒计时
- 展示活动期赠送积分（1.5倍）

## 六、注意事项

### 6.1 活动边界

- 活动仅影响充值赠送积分倍率
- 不影响会员体系核心逻辑
- 活动结束后自动恢复1.0倍
- 活动配置通过后台管理，无需改代码

### 6.2 数据隔离

- 活动配置独立于会员体系
- 充值记录增加 activity_code 字段
- 便于后续活动效果分析

### 6.3 扩展性

- 活动框架设计支持后续其他活动类型
- 如：节日活动、限时折扣、积分翻倍等
- 通过 activity_type 区分活动类型
