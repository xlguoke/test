## 0. 用户登录与认证 [🟢 一期] [新增]

### 0.1 微信授权登录 [H5-AUTH-001]
- H5 端通过微信 OAuth 2.0 静默授权获取 openid [H5-AUTH-001-01]
- 首次授权流程：用户进入 H5 → 微信 redirect → 同意授权 → 回调获取 code → 服务端换取 openid → 创建/查询用户 → 下发 JWT Token [H5-AUTH-001-02]
- 非首次授权：Token 有效则直接进入首页，Token 过期则自动续期（refresh token）[H5-AUTH-001-03]
- 授权失败降级：弹窗提示"授权失败，请重试"，提供"重新授权"按钮 [H5-AUTH-001-04]

### 0.2 微信用户信息获取 [H5-AUTH-002]
- 登录后通过 `wx.getUserInfo` 获取微信昵称和头像（需用户主动触发）[H5-AUTH-002-01]
- 未授权用户信息时使用默认头像 + "微信用户"作为默认昵称 [H5-AUTH-002-02]
- 头像昵称可在会员中心→用户资料中修改（见 §6.8）[H5-AUTH-002-03]

### 0.3 Token 管理 [H5-AUTH-003]
- JWT Token 存储于 localStorage，每次请求通过 Header `Authorization: Bearer <token>` 携带 [H5-AUTH-003-01]
- Token 过期时间 7 天，过期前自动使用 refresh_token 续期 [H5-AUTH-003-02]
- 退出登录：清除本地 Token + localStorage 数据 → 回到首页（访客模式，可浏览商品但不可下单）[H5-AUTH-003-03]

### 0.4 登录态校验 [H5-AUTH-004]
- 以下操作需校验登录态，未登录时弹出登录引导弹窗 [H5-AUTH-004-01]：
  - 提交订单 / 支付 / 查看会员中心 / 查看订单 / 充值 / 积分抵扣
- 以下操作无需登录 [H5-AUTH-004-02]：
  - 浏览首页 / 搜索商品 / 查看商品详情 / 添加购物车（购物车存 localStorage）
- 登录引导弹窗："登录后即可下单享会员折扣" + "微信登录"按钮（绿色主题色）[H5-AUTH-004-03]

### 0.5 登录加载与过渡 [H5-AUTH-005]
- 应用启动时显示品牌 Logo + "正在加载..." 启动页（白底 + 居中 Logo），最多 3 秒 [H5-AUTH-005-01]
- 登录中显示全屏 loading："微信登录中…"，禁止操作 [H5-AUTH-005-02]
- Token 校验完成前使用骨架屏展示首页框架 [H5-AUTH-005-03]

## 28. 后管权限码规划 [🟢 一期] [新增]

> **设计原则**：按钮/操作级权限通过权限码（Permission Code）动态控制。代码层面通过 `v-permission` 指令 + 后端中间件校验，不硬编码角色判断。超级管理员可为任意角色配置权限码组合。

### 28.1 权限码清单 [ADM-PERM-001]

格式：`{模块}:{操作}`

| 权限码 | 模块 | 操作说明 | 默认分配 |
|:---|:---|:---|:---|
| `dashboard:view` | Dashboard | 查看首页面板 | 所有角色 |
| `goods:view` | 商品管理 | 查看商品列表和详情 | 所有角色 |
| `goods:create` | 商品管理 | 新建商品 | 超级管理员 |
| `goods:edit` | 商品管理 | 编辑商品 | 超级管理员 |
| `goods:delete` | 商品管理 | 删除商品 | 超级管理员 |
| `goods:enable` | 商品管理 | 上架/下架商品 | 超级管理员 |
| `order:view` | 订单管理 | 查看订单列表和详情 | 所有角色 |
| `order:make` | 订单管理 | 开始制作（待取餐→制作中） | 超级管理员 |
| `order:complete` | 订单管理 | 完成订单（制作中→已完成） | 超级管理员 |
| `order:cancel` | 订单管理 | 取消订单 | 超级管理员 |
| `order:refund` | 订单管理 | 退款处理 | 超级管理员 |
| `member:view` | 会员管理 | 查看会员列表和详情 | 所有角色 |
| `member:point_adjust` | 会员管理 | 积分调整 | 超级管理员 |
| `member:balance_adjust` | 会员管理 | 余额调整 | 超级管理员 |
| `account:view` | 账号管理 | 查看账号列表 | 所有角色 |

| `log:view` | 操作日志 | 查看操作日志和详情 | 超级管理员 |
| `table:view` | 桌号管理 | 查看桌号列表 | 所有角色 |
| `table:manage` | 桌号管理 | 新增/删除/启用停用桌号 | 超级管理员 |

| `config:edit` | 会员配置 | 编辑/重置会员配置 | 超级管理员 |
| `permission:manage` | 权限管理 | 为角色分配/修改权限码 | 超级管理员 |

### 28.2 权限码使用约定（代码层面） [ADM-PERM-002]

1. **存储**：`admin_users` 表 `permissions` 字段为 `Array<String>`，存储该账号拥有的权限码列表 [ADM-PERM-002-01]
2. **前端校验**：按钮级 `v-permission="'goods:edit'"` 指令控制元素是否渲染；路由守卫校验 `meta.permission` [ADM-PERM-002-02]
3. **后端校验**：JWT 中间件解析 `permissions`，云函数入口校验当前请求是否包含所需权限码，403 拒绝 [ADM-PERM-002-03]
4. **粒度**：操作按钮级（非页面级），未授权按钮不渲染（而非渲染后禁用，防止绕过）[ADM-PERM-002-04]
5. **可配置**：超级管理员在「系统管理→角色权限」页为任意角色自由勾选权限码，修改后立即生效 [ADM-PERM-002-05]
6. **数据状态约束**：权限码判断后可操作，但部分操作仍受数据状态约束（如：有进行中订单的商品不可删除），此为业务规则非权限规则 [ADM-PERM-002-06]

### 28.3 批量操作权限 [ADM-PERM-003]
- 批量操作的权限与对应单项操作一致（如批量上架需 `goods:enable`）[ADM-PERM-003-01]

## 异常与边界处理约定（全系统通用）

> 各模块描述正常流程时省略重复的异常处理，以下约定覆盖全系统通用异常场景。

### 网络异常处理
| 场景 | H5 处理 | 后管处理 |
|:---|:---|:---|
| 请求超时（>10s） | Toast "网络请求超时，请重试" + 页面保留数据不跳转 | Message.error "请求超时" |
| 网络断开 | Toast "网络连接失败，请检查网络" + 离线提示条 | Message.error + 页面顶部红色离线横幅 |
| 网络恢复 | 自动重连 + Toast "网络已恢复"（绿色） | 自动重连 + 隐藏离线横幅 |
| 重试按钮 | 每个操作失败后 3 秒内显示"重试"按钮，点击重新发起请求 | 操作失败弹窗提供"重试"按钮 |

### 服务端异常处理
| 场景 | 处理方式 |
|:---|:---|
| 500 服务器错误 | Toast "服务器繁忙，请稍后重试" |
| 401 Token 过期 | 自动尝试 refresh_token，失败则跳转登录页 |
| 403 无权限 | H5：Toast + 回退；后管：页面展示"403 无权访问" |
| 404 资源不存在 | Toast + 返回上一页 |
| 429 频率限制 | Toast "操作过于频繁，请稍后重试"（5s 内屏蔽重复点击） |

### 并发冲突处理
| 场景 | 处理方式 |
|:---|:---|
| 库存不足 | 提交订单失败 → Toast "XX 商品库存不足" → 返回购物车刷新 |
| 余额不足 | 支付失败 → 自动切换支付方式 |
| 订单状态冲突 | 操作失败 → Toast "订单状态已变更，请刷新" → 自动刷新列表 |
| 重复提交 | 前端防抖 300ms + 服务端幂等校验（相同订单号 10s 内去重） |

### 数据边界处理
| 场景 | 处理方式 |
|:---|:---|
| 列表为空 | 统一空状态组件：插图 + "暂无数据" + （可选）操作按钮 |
| 数据过期 | 列表页进入时检查缓存时间 > 60s → 自动下拉刷新 |
| 图片加载失败 | 显示默认占位图（灰色图标 + 文字） |
| 金额计算精度 | 服务端统一 `Math.round` 至分（整数），前端展示时 `/100` 保留两位小数 |

### 异常恢复机制
| 场景 | 处理方式 |
|:---|:---|
| 支付中断恢复 | 从支付页退出后再进入 → 自动查询订单状态 → 已支付跳转成功页 / 未支付继续支付 |
| 页面崩溃 | H5 端 localStorage 缓存表单草稿（3 分钟），重新进入提示"是否恢复未完成的订单？" |
| 定时任务失败 | 超时取消订单定时任务失败 → 记录 `admin_operation_logs` + 钉钉/企微告警 |

### 全局异常日志
- H5 端：`console.error` + 可选的 Sentry / uniCloud 前端日志上报 [ERROR-HOOK-001]
- 服务端：所有未捕获异常统一记录 `error_logs` 集合（timestamp / endpoint / error / stack / user_id / request_id）[ERROR-HOOK-002]
- 生产环境不向前端暴露 `error.stack`，仅返回统一错误码和友好提示 [ERROR-HOOK-003]
