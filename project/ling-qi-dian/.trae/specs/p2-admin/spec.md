# Spec: 后台管理前端（vben-admin）

## Goal
实现机车俱乐部后台管理系统前端，基于 vben-admin Vue3 框架。包含登录页、Dashboard 首页、商品管理、订单管理、会员管理、系统管理六大模块。所有颜色/字号引用 vben 主题变量或自定义 CSS Variables，禁止硬编码。

## Done Contract
- 完成标准：后台管理所有模块可用，Mock 数据正常加载，权限控制生效
- 证明方式：浏览器访问 admin 端，完成登录 → 查看 Dashboard → 管理商品 → 处理订单 → 查看会员 → 管理系统账号
- 未完成标准：页面无法访问、权限失效、硬编码样式

## Scope
- **In**: 登录页、Dashboard、商品管理（列表/编辑）、订单管理（列表/详情）、会员管理（列表/详情）、系统管理（账号/角色）
- **Out**: H5 前端、服务端 API（在各自独立 spec 中）

## Facts / Constraints
- **产品基准**：`full-feature-list.md` v3.0 §17~§26（ADM-LOGIN-001 ~ ADM-CONFIG-001）
- 框架：vben-admin Vue3（Ant Design Vue 组件库）
- 主题：默认主题，主色对齐 H5 主题色 #4ECDC4
- 布局：左侧菜单 + 顶部导航 + 内容区 + 页脚
- 权限：super（全部功能）/ admin（仅查看，不可管理账号）
- 数据：基于 Mock API（admin-login、admin-goods-list、admin-orders-list 等）
- **设计契约文件**（字段定义、接口结构的唯一来源，禁止猜测）：
  - 接口字段 → [api-contracts.md](../../../mydocs/api-contracts.md) §8 后台管理模块（admin-login / admin-goods-list+save / admin-orders-list+update / admin-members-list+detail+point-adjust / admin-refund-process / admin-report-dashboard / admin-operation-logs / admin-table-list+save）
  - 会员规则 → [member-system-design.md](../../../mydocs/member-system-design.md) §一（等级体系：7 级等级名、累计充值门槛、积分倍率）

## 模块详细规范

### 1. 登录页（views/login/index.vue）

#### 1.1 页面布局
- **左侧**：品牌图（机车俱乐部 Logo + 名称），占 60% 宽度
- **右侧**：登录表单，占 40% 宽度，垂直居中

#### 1.2 登录表单
- **标题**："欢迎回来"（font-size-2xl, font-weight-bold）
- **账号输入框**：
  - placeholder: "请输入账号"
  - 前缀：用户图标
  - 最大 50 字符
  - 圆角 radius-md
- **密码输入框**：
  - placeholder: "请输入密码"
  - 前缀：锁图标
  - 后缀：眼睛图标（切换显示/隐藏）
  - 圆角 radius-md
- **记住我**：复选框，默认勾选
  - 勾选后 7 天免登录（localStorage 存 token）
- **登录按钮**：
  - 全宽，高度 48px，圆角 radius-md
  - 背景 var(--color-primary)，文字白色
  - loading 状态：loading 图标 + "登录中..."
  - 禁用二次点击
- **登录成功**：存储 JWT Token 到 localStorage，跳转 Dashboard
- **登录失败**：账号/密码错误（输入框下方红色文字）
- **表单校验**：账号非空、密码非空、密码长度 ≥6

### 2. Dashboard 首页（views/dashboard/index.vue）

#### 2.1 数据概览卡片
- **布局**：4 列等分，gap: var(--spacing-lg)
- **卡片**：白色背景，圆角 radius-xl，阴影 shadow-md，padding: var(--spacing-lg)
- **内容**：
  1. 今日订单数：数值（font-size-3xl, font-weight-bold）+ 较昨日变化百分比（绿色↑/红色↓）
  2. 今日营业额：同上格式
  3. 新增会员：同上格式
  4. 总会员数：同上格式
- **图标**：卡片左上角，40px，圆角 radius-md，背景主题色浅

#### 2.2 近 7 日营业额趋势图
- **容器**：白色卡片，圆角 radius-xl，padding: var(--spacing-lg)
- **标题**："近 7 日营业额趋势"（font-size-lg, font-weight-bold）
- **图表**：ECharts 折线图
  - X 轴：日期（MM-DD）
  - Y 轴：金额（¥）
  - hover 显示具体金额
  - 线条颜色：var(--color-primary)
  - 填充颜色：var(--color-primary-light)，透明度 0.3

#### 2.3 会员等级分布饼图
- **容器**：白色卡片，圆角 radius-xl，padding: var(--spacing-lg)
- **标题**："会员等级分布"（font-size-lg, font-weight-bold）
- **图表**：ECharts 饼图
  - 7 个扇区，颜色对应等级
  - 图例右侧排列
  - hover 显示数量和百分比

#### 2.4 最近订单表格
- **容器**：白色卡片，圆角 radius-xl，padding: var(--spacing-lg)
- **标题**："最近订单"（font-size-lg, font-weight-bold）
- **表格**：BasicTable，最近 10 条
  - 列：订单号、会员昵称、商品摘要、实付金额、状态、时间
  - 状态标签：待取餐(橙)/已完成(绿)/已取消(红)
  - "查看更多" 链接：跳转订单管理

#### 2.5 快捷入口
- **布局**：4 个按钮，横向排列
- **按钮**：图标 + 文字，圆角 radius-md，边框 1px solid var(--color-border)
- **入口**：商品管理、订单管理、会员管理、社区管理（二期预留）

### 3. 商品管理

#### 3.1 商品列表页（views/goods/list.vue）

##### 搜索区域
- **布局**：BasicForm，横向排列
- **字段**：
  - 商品名称：输入框
  - 分类：下拉选择（咖啡/特调/精酿/小食）
  - 状态：下拉选择（上架/下架）
- **按钮**："查询"（主题色）+ "重置"（默认）

##### 操作按钮
- "新增商品"：主题色按钮，跳转商品编辑页

##### 商品表格
- **组件**：BasicTable
- **列**：
  - 勾选框
  - 图片：60x60px，圆角 radius-sm
  - 名称：font-size-sm
  - 分类：标签样式
  - 价格区间：font-size-sm
  - 库存：font-size-sm，库存 <10 时红色高亮
  - 状态：Switch 开关
    - 上架：主题色
    - 下架：灰色
  - 标签：多个标签横向排列
  - 操作：编辑（主题色链接）+ 删除（红色链接）
- **分页**：底部分页器，10/20/50 条每页
- **排序**：表头点击排序

#### 3.2 商品编辑页（views/goods/edit.vue）

##### 页面布局
- **左侧**（2/3）：表单区
- **右侧**（1/3）：预览区

##### 商品信息表单
- **名称**：输入框，必填
- **分类**：下拉选择
- **描述**：多行文本输入
- **图片上传**：
  - 最多 5 张
  - 上传组件：拖拽或点击上传
  - 图片预览：80x80px，可删除
- **标签**：多选，可自定义输入
- **状态**：Switch 开关

##### SKU 规格管理
- **布局**：动态表格
- **列**：规格名称、价格、库存、操作
- **操作**：添加行（+ 按钮）、删除行（红色删除图标）
- **表单校验**：至少 1 个 SKU、价格 >0、库存 ≥0

##### 预览卡片
- **实时预览**：商品卡片效果
- 图片、名称、价格、标签

##### 操作按钮
- **保存**：校验后提交，成功返回列表页
- **取消**：返回列表页

### 7. 会员配置管理

#### 7.1 等级权益配置（views/members/level-config.vue）

##### 等级权益表格
- **组件**：BasicTable
- **列**：
  - 等级名称：Tag 标签（颜色对应等级）
  - 升级门槛：累计充值金额（元）
  - 饮品折扣：如 9.8 折
  - 工时费折扣：如 9.5 折
  - 配件折扣：如 9.8 折
  - 积分倍率：如 1.1x
  - 洗车次数：每月免费次数
  - 保养折扣：如 9.5 折
  - 救援公里数：如 50km
  - 代步车天数：如 3 天
  - 专属客服：是/否
  - 生日券：数量
  - 操作：编辑（主题色链接）

##### 编辑弹窗
- **表单字段**：
  - 饮品折扣：InputNumber，0.01~1.0，步长 0.01
  - 工时费折扣：InputNumber，0.01~1.0，步长 0.01
  - 配件折扣：InputNumber，0.01~1.0，步长 0.01
  - 积分倍率：InputNumber，1.0~3.0，步长 0.1
  - 洗车次数：InputNumber，0~30，步长 1
  - 保养折扣：InputNumber，0.01~1.0，步长 0.01
  - 救援公里数：InputNumber，0~500，步长 10
  - 代步车天数：InputNumber，0~30，步长 1
  - 专属客服：Switch 开关
  - 生日券：InputNumber，0~10，步长 1
- **保存**：调用 `admin-member-levels-save` API
- **注意**：等级名称和升级门槛为只读，不可修改

#### 7.2 充值档位配置（views/members/recharge-config.vue）

##### 充值档位表格
- **组件**：BasicTable
- **列**：
  - 档位ID
  - 充值金额（元）
  - 赠送积分
  - 成长值
  - 可达等级：Tag 标签或 "—"
  - 启用状态：Switch 开关
  - 操作：编辑（主题色链接）

##### 编辑弹窗
- **表单字段**：
  - 充值金额：InputNumber，单位：元，最小 1
  - 赠送积分：InputNumber，默认 = 充值金额（1:1:1模型）
  - 成长值：InputNumber，默认 = 充值金额（1:1:1模型）
  - 启用状态：Switch 开关
- **保存**：调用 `admin-recharge-tiers-save` API
- **注意**：1:1:1 模型下赠送积分和成长值默认与充值金额一致，可手动调整

## ADDED Requirements

### Requirement: 登录认证
The system SHALL provide secure login with JWT token and role-based access control.

#### Scenario: 登录成功
- **GIVEN** 用户输入正确账号密码
- **WHEN** 点击登录
- **THEN** 存储 Token，跳转 Dashboard

#### Scenario: 权限控制
- **GIVEN** admin 角色用户访问账号管理
- **WHEN** 页面加载
- **THEN** 显示 403 无权访问

### Requirement: 商品状态切换
The system SHALL allow toggling goods status with immediate effect.

#### Scenario: 下架商品
- **GIVEN** 用户点击 Switch 开关
- **WHEN** 状态切换为下架
- **THEN** 商品在 H5 端不再显示
