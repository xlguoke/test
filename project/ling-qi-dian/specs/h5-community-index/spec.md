# h5-community-index Spec v5.0

> 动态广场前端规格
> 版本: v5.0 | 日期: 2026-05-23

---

## 1. 目标

提供社区动态广场页面，支持瀑布流浏览帖子、点赞、图片预览、下拉刷新、上拉加载。

---

## 2. 页面路由

`/pages/community/index`（或项目对应路由）

---

## 3. 数据流

- **入口**：页面 onLoad / onShow 调用 `community-post-list`（type=all, page=1）
- **点赞**：调用 `community-post-like`，成功后局部更新帖子卡片状态
- **下拉刷新**：重新请求 page=1，重置列表
- **上拉加载**：page+1 追加列表

---

## 4. 交互规格

### 4.1 瀑布流布局

- 两列布局，左右列高度自适应
- 每张图片保持原始宽高比，宽度 = (屏幕宽 - 间距) / 2
- 列间距 8px，行间距 8px
- 帖子卡片含：作者头像+昵称、图片（1~9张，九宫格缩略）、文字内容（最多2行，超出省略）、点赞数+评论数、点赞按钮

### 4.2 点赞动画

- 触发：点击心形图标
- 动画：scale 1.0 → 1.3 → 1.0，时长 200ms
- 颜色：未点赞时灰色 #999，点赞后红色 #FF4D4F
- 逻辑：先调接口，接口成功后播放动画并更新状态；接口失败不播放动画，toast 提示

### 4.3 图片预览

- 触发：点击帖子九宫格中的任意图片
- 行为：全屏预览，支持左右滑动切换
- 数据来源：该帖子的 `images` 数组
- 当前索引：点击的第几张，预览即从该索引开始
- 关闭：点击关闭按钮或下滑退出

### 4.4 下拉刷新

- 触发：手指下拉页面
- 视觉：顶部显示 loading 动画（旋转图标 + "刷新中..."）
- 行为：松手后触发 `community-post-list` page=1
- 结束：请求完成后隐藏 loading，列表重置

### 4.5 上拉加载更多

- 触发：滚动到底部
- 视觉：底部显示 loading 动画（"加载中..."）
- 行为：请求 page+1，追加到列表
- 无更多数据：后端返回 list 为空时，显示 "没有更多了"

---

## 5. 状态管理

- 列表数据：本地 state
- 分页状态：page, pageSize, hasMore
- 点赞状态：按 post_id 映射 is_liked（与后端同步）

---

## 6. 错误处理

- 网络错误：toast "网络异常，请稍后重试"
- 接口非 0：toast 后端 message
- 空列表：展示空状态插图 + "暂无动态，去发布一条吧"

---
