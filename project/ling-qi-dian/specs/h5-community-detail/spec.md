# h5-community-detail Spec v5.0

> 帖子详情页前端规格
> 版本: v5.0 | 日期: 2026-05-23

---

## 1. 目标

展示帖子完整内容、嵌套评论列表，支持发表评论、点赞、删除帖子。

---

## 2. 页面路由

`/pages/community/detail?post_id={post_id}`

---

## 3. 数据流

- **入口**：onLoad 获取 `post_id` → 并行调用 `community-post-detail` + `community-comment-list`
- **点赞**：调用 `community-post-like` → 更新页面点赞状态和计数
- **评论**：调用 `community-comment-create` → 成功后清空输入框，刷新评论列表
- **删除**：作者/管理员显示删除按钮 → 点击确认弹窗 → 调用 `community-post-delete` → 成功返回列表页

---

## 4. 交互规格

### 4.1 帖子内容区

- 作者头像 + 昵称
- 发布时间（相对时间或绝对时间）
- 帖子文字内容
- 图片九宫格（点击可预览，同广场页）
- 路线信息卡片（如有）：起点→终点，距离 xx km，时长 xx 分钟
- 点赞数、评论数
- 点赞按钮（心形图标 + 动画，同广场页）

### 4.2 删除帖子

- 仅当当前用户 uid == 作者 uid 时，显示右上角 "删除" 按钮
- 管理员角色也显示删除按钮（若前端能获取角色信息）
- 点击弹出确认弹窗："确定删除该帖子？"
- 确认后调用 `community-post-delete`
- 成功：toast "已删除"，返回上一页
- 失败：toast 错误信息

### 4.3 评论列表

- 嵌套展示：
  - 一级评论：头像、昵称、内容、时间
  - 二级回复：缩进展示，格式 "@被回复者昵称 内容"
- 每条评论显示相对时间（如 "2小时前"）
- 列表按时间升序排列
- 点击评论可回复（二期功能，一期仅展示）

### 4.4 发表评论

- 底部固定输入栏：
  - 默认显示输入框 + "发送" 按钮
  - 点击输入框展开全屏输入（或键盘弹起）
- 输入框 placeholder："说点什么..."
- 发送按钮：内容为空时 disable，有内容时 enable
- 发送中：按钮显示 "发送中..."，禁用
- 发送成功：清空输入框，收起键盘，刷新评论列表，帖子评论数 +1
- @功能：二期实现，一期预留

---

## 5. 状态管理

- 帖子详情：本地 state
- 评论列表：本地 state + 分页
- 点赞状态：本地 state，与后端同步
- 当前用户身份：从全局用户状态获取

---

## 6. 错误处理

- 帖子不存在：展示空状态 "帖子已删除或不存在"，提供返回按钮
- 网络错误：toast "网络异常，请稍后重试"
- 评论发送失败：toast 错误，保留输入内容

---

## 7. Done Contract

- [ ] 帖子详情完整展示，含路线信息卡片
- [ ] 评论列表嵌套展示正确，一级+二级区分清晰
- [ ] 点赞状态切换实时更新
- [ ] 发表评论流程完整，输入框交互顺畅
- [ ] 删除帖子确认弹窗 + 权限控制正确
