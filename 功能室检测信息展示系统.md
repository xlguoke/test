# 功能室检测信息展示系统

## 一、项目概述

**功能室检测信息展示系统**是一个面向检验检测实验室的综合性智慧屏管理系统，支持13种不同类型的屏幕终端，包括坐式屏、设备小屏、功能室大屏、功能室门口屏、标准物质间、化学药品室、标准养护室、样品室、留样室、外检室、外检室数据大屏、室内屏以及指挥中心驾驶舱等。

该系统通过统一的授权绑定机制，实现了实验室各类功能室的信息可视化展示、设备管理、任务追踪、温湿度监控、智能送样管理等核心功能，为实验室数字化转型提供了完整的解决方案。

## 三、核心功能模块

### 3.1 功能室操作屏 2.0（functionRoom2）

功能室操作屏是系统的核心模块，采用模块化设计，支持动态配置渲染。

#### 3.1.1 首页组件系统

首页采用**动态模板渲染机制**，通过后端配置控制展示内容：

```typescript
// 支持的组件板块枚举
enum WidgetList {
  LabIntro = "LabIntro",              // 功能室简介
  Task = "Task",                       // 任务模块
  HumitureInfo = "HumitureInfo",       // 温湿度信息
  SampleInfo = "SampleInfo",           // 送样管理
  DeviceInfo = "DeviceInfo",           // 设备信息
  EqOutApply = "EqOutApply",           // 设备外出申请
  StandardRoom = "StandardRoom",       // 标养室
  ExternalCheckRoom = "ExternalCheckRoom", // 外检室
  StandardSubstanceRoom = "StandardSubstanceRoom", // 标准物质间
  SampleRoom = "SampleRoom",           // 样品室
  SampleStorageRoom = "SampleStorageRoom", // 留样室
  MedicineRoom = "MedicineRoom"        // 药品室
}
```

**动态渲染逻辑**：
- 从后端获取模板配置 `templateConfig`
- 解析 `renderWidgets` 配置数组
- 根据 `key` 映射到对应组件进行渲染
- 支持自定义组件（`CUSTOM` 类型）

#### 3.1.2 温湿度实时监控

- **数据来源**：物联网平台 WebSocket 实时推送
- **实现类**：`IotDataRequest` 类封装了完整的物联网数据交互逻辑
- **功能特性**：
  - 自动获取物联网 Token
  - WebSocket 连接管理（自动重连、断线处理）
  - 实时温湿度数据展示
  - 房间工作状态智能判断（开启/部分开启/关闭）

```typescript
// 物联网数据请求类核心方法
class IotDataRequest {
  async getIotToken()           // 获取物联网平台 Token
  listenDataUpdate()            // 监听数据更新
  closeWebSocket()              // 关闭连接
}
```

设备管理模块采用**左右分栏布局**：

### 3.6 大厅座式屏（hallSeatScreen）

### 4.1 屏幕适配方案

系统支持多种分辨率屏幕，采用**动态 rem 适配方案**：

```typescript
// 初始化字体大小（基于 1920px 设计稿）
const initfontSize = () => {
  const winW = document.body.clientWidth || window.innerWidth
  let size = (winW / 1920) * 100
  document.documentElement.style.fontSize = size + "px"
}
```

### 4.2 屏幕配置管理

```typescript
// 全局状态管理
export const useStore = defineStore("screenConfig", {
  state: () => ({
    screenConfig: {} as IScreenConfig,  // 屏幕配置
```

### 4.3 数据刷新机制

封装 `useScreenHooks` 实现定时数据刷新：

```typescript
export const useScreenHooks = () => {
  function refreshData(callback: () => void) {
    setTimeout(() => {
      callback && callback()
      refreshData(callback)
    }, screenConfig.value.frequencyRefresh * 1000)
  }
}
```
